# Neovim Config

Personal Neovim configuration built around Neovim's built-in package manager (`vim.pack`) and native LSP config (`vim.lsp.enable`).

## Structure

- `init.lua` - entry point that loads all config modules.
- `lua/config.lua` - editor options.
- `lua/plugins.lua` - plugin installation and plugin setup.
- `lua/keymaps.lua` - global keymaps with `desc` metadata for `which-key`.
- `lua/autocmd.lua` - general autocommands.
- `lua/treesitter.lua` - Treesitter startup and configured parser installer command.
- `lua/lsp.lua` - LSP enablement, diagnostics, and buffer-local LSP keymaps.
- `lsp/*.lua` - native LSP server definitions.
- `nvim-pack-lock.json` - lock file generated by `vim.pack`.
- `stylua.toml` - Lua formatting rules.

## Main Features

- Native Neovim LSP setup without `nvim-lspconfig`.
- Declarative Mason tool installation through `mason-tool-installer.nvim`.
- Completion through `blink.cmp`.
- Formatting through `conform.nvim`.
- Treesitter highlighting, folds, and indentation.
- File tree through `nvim-tree.lua` with custom `h`/`l` navigation.
- Fuzzy finding and LSP lists through `fzf-lua`.
- Git signs and line blame through `gitsigns.nvim`.
- Keymap discovery through `which-key.nvim` using the LazyVim-like `helix` layout.
- UI enhancements through `snacks.nvim`, `noice.nvim`, `lualine.nvim`, and `gruvbox-material`.

## Mason Tools

The following tools are installed automatically on Neovim startup if missing:

- `lua-language-server`
- `vtsls`
- `css-lsp`
- `stylua`
- `prettier`

## LSP Servers

- `lua_ls` for Lua.
- `vtsls` for JavaScript, TypeScript, JSX, and TSX.
- `cssls` for CSS, SCSS, and Less.

LSP server configs live in `lsp/` and are enabled from `lua/lsp.lua`.

## Formatting

Formatting is handled by `conform.nvim`.

Configured formatters:

- Lua: `stylua`
- JavaScript/TypeScript/React: `prettier`
- JSON/JSONC: `prettier`
- CSS/HTML/Markdown/Astro/YAML: `prettier`

Format on save is enabled with `lsp_format = "fallback"`.

## Treesitter

Treesitter parsers are not installed on every startup. To install configured parsers manually, run:

```vim
:TSInstallConfigured
```

The command installs the parser list from `lua/treesitter.lua`.

## Keymaps

Leader key: `<Space>`

Common mappings:

- `<leader><leader>` - find files.
- `<leader>/` - live grep.
- `<leader>fb` - buffers.
- `<leader>fr` - recent files.
- `<leader>fh` - help tags.
- `<leader>fd` - document diagnostics.
- `<leader>e` - toggle file tree.
- `<leader>xq` - quickfix list.
- `<leader>xd` - document diagnostics.
- `<leader>gg` - LazyGit through Snacks.
- `<leader>ub` - toggle Git current line blame.
- `<leader>n` - notification history.
- `<C-s>` - save file.
- `<Esc>` - clear search highlight.

LSP mappings are buffer-local and only available after an LSP client attaches:

- `gd` - go to definition.
- `gr` - references through `fzf-lua`.
- `<leader>ss` - document symbols.
- `<leader>fo` - format current buffer.
- `<leader>uh` - toggle inlay hints.

`which-key.nvim` reads descriptions directly from `vim.keymap.set(..., { desc = ... })`, so keymap descriptions should live with the keymaps rather than being duplicated in plugin setup.
